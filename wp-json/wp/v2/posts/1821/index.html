If you have a website containing 100s of pages, chances are you will have pages which have been deleted or not existing. Somehow these pages are still appearing and people visit these pages from referral sites.<\/p>\n

This results in a decrease in your organic rankings and hence provides a way to your competitors to rank. In this article, I will teach you how to track 404 errors with Google Tag Manager and view the reports in Google Analytics.<\/p>\n

Step1: Check the error message<\/strong><\/h2>\n

The very first step you will have to do is to check what message displays when a nonexisting page is loaded. On some websites, the title displays as \u2018404 not found\u2019  and some websites display simply the website name and message in the body.<\/p>\n

You can test this by simply typing in the URL, www.yourwebsite.com\/test<\/a>. This will display not found page on your website.<\/p>\n

404 not found in the title is easy to track so I will explain the second method. There is no 404 not found message displayed in the body nor the title. What will you do?<\/p>\n

In this case, you will have to right click and open page source. Press ctrl+f and type not found, this will display the id or CSS selector.<\/p>\n

<\/p>\n

Copy this ID<\/strong> and go to your GTM account.<\/p>\n

Step2: Create a User-Defined Variable<\/strong><\/h2>\n

Open your Google Tag Manager account and navigate to variables section.<\/p>\n

Create a new user-defined variable<\/strong>.<\/p>\n

Select the variable type as DOM element
\n<\/strong><\/p>\n

Select selection method as CSS Selector
\n<\/strong><\/p>\n

Enter element selector as body
\n<\/strong><\/p>\n

Enter id<\/strong> in the attribute name field<\/p>\n

Enter a name of the variable and hit save<\/strong><\/p>\n

<\/p>\n

Step3: Create a new Trigger<\/strong><\/h2>\n

Navigate to triggers section and create a new trigger
\n<\/strong><\/p>\n

Select the trigger type as DOM ready
\n<\/strong><\/p>\n

Select the trigger to fire on some DOM ready events
\n<\/strong><\/p>\n

Select the not found variable equals not-found<\/strong>
\nEnter the trigger name as not found<\/strong><\/p>\n

<\/p>\n

Hit save<\/strong><\/p>\n

Step4: Create a New Tag<\/strong><\/h2>\n

Now, we have to send this data to Google Analytics. To do this, we will create a new universal analytics tag.<\/p>\n

Select the tag type as Event
\n<\/strong><\/p>\n

Enter the tag name as GA Event PageNotFound<\/strong>
\nCategory: Page Not Found<\/strong><\/p>\n

Action: {{Page Path}}<\/strong><\/p>\n

Label: {{Referrer}}<\/strong>
\nNon-Interaction Hit: True<\/strong><\/p>\n

<\/p>\n

Add the trigger we created in step 3.<\/p>\n

Preview and publish the once you make sure it\u2019s firing correctly.<\/p>\n

Conclusion<\/strong><\/h2>\n

The biggest benefit of tracking 404 pages is you can resolve the issues whenever it arises. This tracks even faster than search console and lets you find the issues instantly.
