Recently I came across a website having a single domain with multiple extensions. By extensions, I mean the hostname was single. For example, for a website www.abc.com, the extensions were www.abc.co.uk, www.abc.fr, www.abc.mx and etc.<\/p>\n

The website was same but it has multiple clones for different locations and languages. The objective was to track all these extensions within a single analytics account.<\/p>\n

If you have this kind of website, you will have to setup cross domain tracking. This was an woocommerce website and we have to setup enhanced <\/a>ecommerce<\/a> tracking via GTM<\/a>. <\/p>\n

For this kind of website, you will have a single GTM container and single GA property. Add your GTM tracking code to the main domain that is www.abc.com.<\/p>\n

Setting up cross domain tracking is quite easy in GTM. let\u2019s go through a step by step guide to help you understand easily.<\/p>\n

Step1: Setup GTM & GA<\/strong><\/h2>\n

As mentioned above, you will have to setup GTM tracking code on the main website. You will then connect GA with GTM using the pageview tag.<\/p>\n

Add all the GA tags that you want to use for sending ecommerce of event data to analytics. Make sure the data is appearing in Google Analytics.<\/p>\n

Step2: Setup Cross Domain Tracking<\/strong><\/h2>\n

To setup cross domain tracking, you will have to configure all the GA tags. For example, to track pageviews on all extensions, open the GA pageview tag.<\/p>\n

Click on more settings<\/p>\n

Click on fields to set<\/p>\n

Add the two fields as shown in the image below<\/p>\n

Do the same thing for all other GA tags you have setup. Preview changes and open each domain extension to verify the tags are firing and data is visible in analytics.<\/p>\n

If everything is ok, save and publish changes.<\/p>\n

Step3: Add All Extensions to Referral Exclusion List<\/strong><\/h2>\n

If you don\u2019t add all the extensions to your referral exclusion list, you will see these extensions listed in referral report. It is important that you always add your domain name plus any third party extension that you are using for transactions i.e paypal.com to the referral exclusion list<\/a>.<\/p>\n

Step4: Create Separate View for Each Extension<\/strong><\/h2>\n

Now, if you want to view data for each extension separately, you will have to create a new view and create a hostname filter<\/a>. This will make sure that the view will only record data from the specific extension and not from all.<\/p>\n
