Scroll Tracking via Google Tag Manager has got a lot easier now!<\/p>\n

If you have a content-based website, you would be interested to figure out if visitors scroll your page. (Want to know what else to track to measure your content performance? Check out this article about growing your content site right<\/a>)<\/p>\n

Google Analytics displays the avg. time on page but there isn\u2019t something built-in that would tell if visitors actually scroll your page or even read the whole article.<\/p>\n

To do this we all have been using a custom HTML tag in Google Tag Manager<\/a> and then create a tag that will send data to analytics. Fortunately, Google Tag Manager has recently published a built-in trigger that automatically listens to page scrolls.<\/p>\n

In this tutorial, I will walk you through a simple step by step process to setup scroll tracking using Google Tag Manager.<\/p>\n

Step1: Enable Scroll Depth Trigger<\/strong><\/h2>\n

To find the native scroll depth trigger, navigate to triggers<\/strong>, click new<\/strong> button to create a new trigger and then select scroll depth<\/strong> from the trigger type.<\/p>\n

<\/p>\n

When you click the scroll depth trigger, you will see two types of scroll depths.<\/p>\n

\u00a0 Vertical scroll depths <\/strong>= track vertical scroll<\/p>\n

\u00a0 Horizontal scroll depths <\/strong>= track horizontal scroll<\/p>\n

When you click any of the methods you will then see two ways to measure the scroll.<\/p>\n

\u00a0 Percentages:<\/strong> track scroll depth by percentage(0,25,50,75,100)<\/p>\n

\u00a0 Pixels: <\/strong>track scroll by pixels(769, 1025, 1441)<\/p>\n

Almost all of the websites scroll vertically, so we will measure the vertical scroll depth. Also, it is easy to under percentages rather than pixels. We will track 0, 25, 50, 75 and 100 percentage.<\/p>\n

The one very important thing to know about this trigger is to select the pages on which you want to fire this trigger. We can configure this trigger to fire on the blog page, but if your website contains multiple categories, you can create a regex for multiple pages.<\/p>\n

<\/p>\n

Once you configure the trigger, click on the save button.<\/p>\n

\n

Step2: Enable Native Scroll Variables <\/strong><\/h2>\n

Google Tag Manager has also introduced datalayer variables for scroll depth. These variables automatically capture the required data from the datalayer. You only have to enable them. To do this, navigate to variables, click configure, and scroll down to see the scrolling built-in variables.<\/p>\n

<\/p>\n

Step3: Create a Scroll Tracking Tag<\/strong><\/h2>\n

Now the final step is to send the scroll data to Google Analytics. To do this, navigate to tags, click new, click tag configuration, and select universal analytics as tag type.<\/p>\n

<\/p>\n

Select the track type as event<\/p>\n

<\/p>\n

Enter the event parameters as displayed in the image below.<\/p>\n

<\/p>\n

While adding event tag parameters, you will have to be careful about the Non-Interaction Hit field.<\/p>\n

You can configure the non-interaction field to be true for 0% threshold. This will make sure your site bounce rate isn\u2019t affected even when the page isn\u2019t scrolled. To do this you can create a lookup variable that will output true when the scroll depth threshold equals to 0%.<\/p>\n

Configure the trigger as displayed in the below image.<\/p>\n

You can also set this up for 25% if you have a small page and threshold value changes to 25% on page load.<\/p>\n

<\/p>\n

Rename this variable and hit save.<\/p>\n

Navigate back to the scroll depth tag and select the lookup table variable from the non-interaction hit drop-down. Your new tag configuration should look similar to the below image.<\/p>\n

<\/p>\n

Click on the triggering option and select the scroll depth trigger we created in the first step.<\/p>\n

Save the tag.<\/p>\n

Note: If you have already setup scroll tracking using the custom HTML tag method, you can simply delete the custom HTML tag and change the scroll tag configuration. <\/strong><\/p>\n

\n

Step4: Preview and Debug Changes<\/strong><\/h2>\n

Before publishing the changes the changes in your container, it is necessary to review and debug changes. Click on the preview button within the GTM container, once the preview mode loads, open your website in a new tab.<\/p>\n

When your website fully loads, scroll down to see if the scroll depth tag is firing. If it fires, you have setup successfully, otherwise go to step one and look if you have missed anything.<\/p>\n

<\/p>\n

If you want to verify data in Google Analytics, open GA account in a new tab and navigate to real time >> events<\/strong>.<\/p>\n
