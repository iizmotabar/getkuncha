How to use the Google Analytics Checklist?<\/strong><\/h3>\n

This checklist is available for everyone and is set to view only mode. To use this checklist you will have to make a copy.<\/p>\n

<\/p>\n

For each item on the list there are three responses:<\/p>\n

Looks good:<\/strong> this means no issues found<\/p>\n

Need Fixes:<\/strong> this is making the data inaccurate and need to fix<\/p>\n

N\/A:<\/strong> this doesn\u2019t affect any data quality<\/p>\n

23 points data quality checklist<\/strong><\/h2>\n

1. Google Analytics Structure<\/strong><\/h3>\n

You should always make sure to have an understandable and well-defined account structure. Clients usually aren\u2019t aware of this and they proceed with the default settings. In a basic Google Analytics account, you should have a property with three different views.<\/p>\n

    \n
  1. All website data view (unfiltered data)<\/li>\n
  2. Master view (filtered data for analysis)<\/li>\n
  3. Test view (for testing filters and goals)<\/li>\n<\/ol>\n

    This makes sure that you have a single view which contains accurate data that has filters and goals added. You can use this to analyze data and make decisions.<\/p>\n

    2. Multiple Analytics Codes\/Multiple Pageviews<\/strong><\/h3>\n

    Your website should always contain a single Google Analytics tracking code because too many cooks spoil the broth. Unless you don\u2019t add multiple codes for some reasons. Before doing anything test with Google Tag Assistant to find the analytics code and if the tracking ID matches the account ID.<\/p>\n

    You can also test with GA debugger to verify only single tracking ID exists on the website. You can add multiple Google Analytics account on the same website but having the same tracking code multiple times<\/a> send duplicate hits.<\/p>\n

    <\/p>\n

    3. Bot filtering<\/strong><\/h3>\n

    Since we are only interested in humans as traffic, you must shut the door of your website to the bots and spiders. Bot filtering is an internal feature of Google Analytics which blocks known bots and spiders. You will have to make sure this is enabled in your filtered view\/master view so you will see the accurate data. You can do this by navigating to admin >> view settings >> bot filtering.<\/p>\n

    <\/p>\n

    4. Spam Filtering<\/strong><\/h3>\n

    Spam traffic and bot traffic are two different things but you have to exclude both from your analytics data. The common way to identify a spam traffic source is by looking at bounce rate, %new sessions,\u00a0and pages\/session. You can follow this guide<\/a> to learn how to identify spam filters and remove them from your analytics data.<\/p>\n

    5. Search Console<\/strong><\/h3>\n

    Connecting search console with Google Analytics allows you see all the SEO data in analytics. You can see the top organic keywords and landing pages. To verify the search console linking, navigate to admin >> property settings >> adjust search console.<\/p>\n

    <\/p>\n

    6. Enable Demographics & Interest Report<\/strong><\/h3>\n

    This is one of the important things that most of the people aren\u2019t aware of and forget to enable it. Enabling demographics allows you to see the visitor\u2019s location, gender, age,\u00a0and interests. A ton of data which can help your analysis process easier.<\/p>\n

    To verify this, navigate to admin >> property settings >> enable demographics and interest reports.<\/p>\n

    <\/p>\n

    Related post: why google analytics not showing up demogaphics reports?<\/a><\/p>\n

    7. Exclude Internal Traffic<\/strong><\/h3>\n

    You should always exclude your internal traffic from being tracked in Google Analytics. It is recommended that you exclude all internal traffic using filters. This way your data will be only related to external website visitors and anyone from within your internal business.<\/p>\n

    You can easily exclude multiple internal users<\/a> with a custom filter.<\/p>\n

    8. Currency & Time Zone<\/strong><\/h3>\n

    By default, the time zone selected in Google Analytics is ‘United States’. Verify that you have added the right time zone so you will see the data at the right time. Secondly, the default currency setup is USD, so you will have to change that according to the currency you are using. You can verify both these settings under view settings.<\/p>\n

    9. Site search tracking<\/strong><\/h3>\n

    Site search is also an important feature you need to enable to be able to get benefit of Google Analytics. This isn\u2019t enabled by default so you will have to enable it after you set up a new account.<\/p>\n

    Site search tracking enables you to track the keywords visitors search on your site. Enabling site search<\/a> helps you track what visitors are searching for on your website. \u00a0To verify, navigate to admin >> view settings >> site search settings.<\/p>\n

    10. Goal Conversion Issues <\/strong><\/h3>\n

    So the next thing you will need to verify is conversion tracking. If you are investing money on Adwords then you will definitely want to see the results. Besides setting up conversions in Adwords, you can also import Google Analytics goals.<\/p>\n

    It is very important to verify that your goals are tracking properly. To verify this, you can simply follow an action on the website to which the goal is linked and check in Google Analytics.<\/p>\n

    11. Bounce rate and overall GA analysis<\/strong><\/h3>\n

    One of the primary metric to monitor in Google Analytics is Bounce Rate<\/a>. There are two cases when you need to worry about. Either the bounce rate is too high or is too low.<\/p>\n

    So if you see the bounce rate to be too high or too low(unexpected), it\u2019s is necessary to diagnose for the problem.<\/p>\n

    12. Referral Exclusion<\/strong><\/h3>\n

    If you are managing an ecommerce website or a website with multiple domains or subdomains, you have to be careful about referral traffic. You will either setup cross domain tracking or sub-domain tracking.<\/p>\n

    Once you setup any of the option according to your website setup, you will have to make sure to add the root domain to referral exclusion list. This excludes root domain from showing up as referral traffic in analytics.<\/p>\n

    To verify, navigate to admin >> property >> tracking code >> \u00a0referral exclusion list. Verify if you have already added the root domain and any other payment gateway you are using on your website.<\/p>\n

    <\/p>\n

    13. Adwords linking<\/strong><\/h3>\n

    If you are running Adwords campaigns for your clients, make sure you have connected the right Adwords account with analytics. This way you will be able to import analytics goals into Adwords and also use analytics data for remarketing<\/a>.<\/p>\n

    Sometimes you link the Google Adwords account but some note set and product campaigns are being recorded in Adwords. This can be because some traffic is sent with google\/cpc or there is another Adwords sending traffic.<\/p>\n

    <\/p>\n

    14. Events (values)<\/strong><\/h3>\n

    Event tracking help you measure actions visitors take on your website. So you it is necessary to confirm that the values you are passing via event are correct and meaningful. If you are tracking multiple events on a site, sometimes it happens that you mistakenly pass different values.<\/p>\n

    For example, you are tracking multiple forms on a website, if you set event category as \u2018Form Submission\u2019 and \u2018form submission\u2019 then you will see two different categories in analytics. The good way is to keep the event category same for similar types of actions.<\/p>\n

    15. Event tracking errors(site errors)<\/strong><\/h3>\n

    Sometimes it happens that due to website code changes or code errors, the event doesn\u2019t fire properly. So you can\u2019t see the data you need in analytics. Verifying the events are executing properly should be the part of data quality test. With Google Tag Manager<\/a> it\u2019s even more easier to track and test event tracking.<\/p>\n

    16. Missing event parameters(undefined, not set)<\/strong><\/h3>\n

    Another issue which occurs after setting up event is missing parameter values. Sometimes you may see undefined or not set. This issue commonly occurs when you setup event tracking via Google Tag Manager<\/a>.<\/p>\n

    17. Channel grouping <\/strong><\/h3>\n

    Google Analytics by default has multiple default channel groups. So all the traffic sources fall into these categories. But if there is any source that doesn\u2019t belong to any of this, you can define a new channel grouping.<\/p>\n

    Sometimes a traffic source shows up in a different channel which makes the data inaccurate.<\/p>\n

    18. Sub or cross-domain requirements<\/strong><\/h3>\n

    If the website is based on multiple domains or have subdomains, you will have to careful about tracking the same user on both sites. To verify cross-domain or subdomain tracking is working correctly, install GA Debugger extension in your browser.<\/p>\n

    Right click inspect and select search console. Copy the client ID appearing in console and then navigate to the other domain or subdomain. If cross-domain or subdomain is setup correctly, the client ID will remain the same, otherwise a new client ID will be generated.<\/p>\n

    <\/p>\n

    19. Frames \/ dynamic pages<\/strong><\/h3>\n

    If there are any iframes added on the website, then you might not be able to track events properly. For this to happen the analytics tracking code should also be added on the iframe.<\/p>\n

    20. Ecommerce tracking<\/strong><\/h3>\n

    If you are dealing with ecommerce website, you have to verify ecommerce tracking is enabled and the ecommerce goals are setup correctly. To do this, navigate to admin >> view >> ecommerce settings.<\/p>\n

    <\/p>\n

    Now, ecommerce tracking isn\u2019t simple as enabling a button. You have to make sure that you add proper ecommerce tracking codes on your site or use the Google Tag Manager plugin for enhanced ecommerce in Woocommerce<\/a>.<\/p>\n

    21. Multiple transactions<\/strong><\/h3>\n

    There may occur multiple issues after setting up enhanced ecommerce tracking and you may not notice because you can see the transaction details. You can create a simple custom report to verify duplicate transactions<\/a>. One of the proper ways to fix duplicate transactions is by setting up cookies with Google Tag Manager<\/a>.<\/p>\n

    22. Enhanced ecommerce steps<\/strong><\/h3>\n

    The biggest benefit of enhanced ecommerce tracking is you can track the purchase behavior. Visitors starting from viewing product to the final purchase. You need to verify that all the steps the steps are working properly and you are not missing data anywhere. The issues arises if a certain event isn\u2019t executing properly and Google Analytics isn\u2019t receiving data for that step.<\/p>\n

    <\/p>\n

    23. Audiences<\/strong><\/h3>\n

    Google Analytics allows you to create an audience and use in adwords for remarketing purposes. When you are importing any audience from Google Analytics, make sure the conditions are set properly and the audience is correct.<\/p>\n
