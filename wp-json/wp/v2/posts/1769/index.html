Installing Hotjar with Google Tag Manager has been easier than using the conventional method of adding Hotjar tracking code to your website. Below is a step by step guide to install Hotjar via google tag manager template.<\/p>\n

\u00a0<\/p>\n

Step1: Setup Your Hotjar Account<\/strong><\/h2>\n

If you don\u2019t have a Hotjar<\/a> account yet, you have to sign up for a free account and add your website. If you already have an account, you can add a new site by clicking on the new site button under sites and organizations.<\/p>\n

<\/p>\n

If you have the Hotjar tracking code on your website, you need to remove it before continuing this.<\/p>\n

\u00a0<\/p>\n

Step2: Get the Hotjar Website ID<\/strong><\/h2>\n

Once you add your website to Hotjar account, you need to copy the unique website ID. You can get this by clicking on the setting dropdown menu and then clicking on sites & organizations.<\/p>\n

Here you will see the list of your websites connected with Hotjar. Copy the website id of the specific website.<\/p>\n

<\/p>\n

\u00a0<\/h2>\n

Step3: Install Hotjar via Google Tag Manager<\/strong><\/h2>\n

1. Login to your tag manager account and select the specific container<\/p>\n

2. Create a new tag and select hotjar as the tag type.<\/p>\n

3. Paste the hotjar site ID you previously copied<\/p>\n

<\/p>\n

4. Create a trigger to fire on specific pages. In my case I use hotjar on all pages, so i am selecting the all pages trigger.<\/p>\n

<\/p>\n

Read Also: How to Install Shareaholic with Google Tag Manager<\/a><\/p>\n

Step4: Test & debug this tag<\/strong><\/h2>\n

Now it\u2019s time to test the changes in the workspace before setting them live. Click on the prevew button to preview the changes and see if the tag is firing successfully.<\/p>\n

<\/p>\n

Leave the preview mode and publish the new workspace changes. Another way is to open your Hotjar account and verify installation inside the dashboard.<\/p>\n
