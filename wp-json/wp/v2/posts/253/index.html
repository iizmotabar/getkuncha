    \n 1. Understanding GA Hierarchy<\/b>
    \n<\/h2>\n

    While setting up a google analytics account for tracking website traffic, you first handedly need to understand the hierarchy and the order in which analytics Accounts, properties and views work.<\/p>\n

    <\/p>\n

    Account:<\/strong><\/p>\n

      \n
    1. You can have up to 100 analytics account under a single google account.<\/li>\n
    2. A single account can have up to 50 properties, which means you can connect upto 50 websites to a single account.<\/li>\n
    3. Each account must contain at least one property to collect your website data<\/li>\n<\/ol>\n

      Property:<\/strong><\/p>\n

        \n
      1. Each property can have up to 25 views.<\/li>\n
      2. If you have a big business you can create multiple properties to get data separately from different devices.<\/li>\n
      3. You can also create multiple properties to direct a specific region or brand data to each property.<\/li>\n<\/ol>\n

        Views: <\/strong><\/p>\n

          \n
        1. This is where you can see traffic data and reports.<\/li>\n
        2. The benefit of creating multiple views is that you can filter data to see reports for different regions or campaigns separately.<\/li>\n
        3. You should always create one unfiltered view, one test view and one master view. The unfiltered view will contain all your website\u2019s raw data, the test view should be setup for testing filters and the master data will have the filtered data that will be used for analyzation.<\/li>\n
        4. \n

          The thing with views is that they disregard any data that was collected before they were created. Each view only includes data that flows in after they are newly created.<\/p>\n<\/li>\n

        5. If you delete a view, you can restore it within the next 30 days. But you should have the admin access to delete or restore any view.<\/li>\n<\/ol>\n

          <\/p>\n

          You can grant permission<\/a> to other users on each level.<\/p>\n

          \u00a0<\/p>\n

          \n 2. Setting up goals<\/b>
          \n<\/h2>\n

          Setting up goals in google analytics should be the most important step after the google analytics installation. Goals in google analytics are used to track conversions. Your website goals may include a newsletter signup or a product purchase.<\/p>\n

          For example, your website goal is to get more newsletter signups. After setting up this goal in analytics, every time a user signs up for newsletter, a goal will be completed. A goal completion is also called a conversion in analytics term.<\/p>\n

          We have already discussed in detail about setting up goals in google analytics<\/a>.<\/p>\n

          \n 3. Excluding Internal Traffic<\/b>
          \n<\/h2>\n

          Google Analytics, in particular, is built for tracking external website visitors. Sometimes, when you or someone in your company visits your website, a hit is counted.<\/p>\n

          This is something that is to be avoided, considering employee traffic can disrupt your analyzation reports as are they are not your actual target audience and they cannot be classified as web natural traffic.<\/p>\n

          So, you will always want to track your target audience and not your employees or yourself. To block all the hits coming from your internal sources, there is a process known as excluding internal traffic.<\/p>\n

          To exclude internal traffic you can follow any of the method below:<\/p>\n

          \u00a0<\/p>\n

          Exclude IP Address<\/strong><\/h3>\n

          To exclude an internal IP address, you have to create a filter. Do you remember in which view to create the filter first? If you don\u2019t know, create the filter in the test view, verify if it works and then copy it to master view. Let\u2019s create the filter in test view.<\/p>\n

          Navigate to analytics admin and select the test view.<\/p>\n

          Under view menu click on filters.<\/p>\n

          <\/p>\n

          Click on Add Filter<\/strong> button<\/p>\n

          From the Filter settings, enter the filter name as \u201cExclude Internal IP\u2019\u201d, then select filter type as \u201cPredefined\u201d. From the filter type drop-down menu, select \u201cExclude\u201d, then from select source or destination menu, select \u201ctraffic from the IP addresses\u201d and finally select expression as \u201cthat are equal to\u201d.<\/p>\n

          Paste the IP address(if you don\u2019t know your IP address, go to google.com and search \u201cwhat is my ip\u201d) in the box underneath and Click save.<\/p>\n

          <\/p>\n

          Once you make sure the filter works<\/a>, copy this filter to the master view. This method is mostly used when you have to exclude a single IP address.<\/p>\n

          For excluding multiple static and dynamic IPs click here<\/a>.<\/p>\n

          \u00a0<\/p>\n

          Install Opt Out Add-on<\/strong><\/h3>\n

          Use this method when you have to exclude multiple IP addresses or all employees from within your company. Install the google analytics opt out addon<\/a> on every browser you want to exclude traffic from. This will automatically exclude any hit coming from internal IPs.<\/p>\n

          \u00a0<\/p>\n

          \n 4. Filter Bots and Spam traffic<\/b>
          \n<\/h2>\n

          The traffic that is coming from web crawlers, bots or any other non-human source is known as spam traffic. Having a huge number of spam traffic can result in inaccurate calculations. To make future business decisions, your data should be accurate and free from any contaminations.<\/p>\n

          Whenever you create a new view, don\u2019t forget to filter bots. To enable this in your analytics view, navigate to admin >> View >> View Settings.<\/p>\n

          This will exclude all the known bots from your website\u2019s actual traffic.<\/p>\n

          Viewing spam traffic<\/strong><\/h3>\n

          In case you ever want to monitor how much spam traffic you site is actually getting, navigate to Acquisition >> All Traffic >> Referrals<\/strong>.<\/p>\n

          <\/p>\n

          Change the date range to at least more than two months.<\/p>\n

          <\/p>\n

          Now sort the reports by bounce rate. Click on the bounce rate column. In the reports, any source with either 0% or 100% bounce rate is a spam traffic.<\/p>\n

          <\/p>\n

          To learn more about eliminating all the spam traffic from your website, read the ultimate guide to getting rid of all the spams in google analytics<\/a>.<\/p>\n

          \n 5. connect Search Console with Google Analytics<\/b>
          \n<\/h2>\n

          Google search console contains google search result data related to your website. This includes your website\u2019s keywords rankings, crawl error, indexing and security updates.<\/p>\n

          You should never forget about this data if you want to stay on top of search results. You would be happy to know that you can view this data in google analytics.<\/p>\n

          Note: Make sure to verify your site with google<\/a>.<\/p>\n

          Let\u2019s move on to directing your search console data to analytics.<\/p>\n

          Sign in to your analytics account<\/p>\n

          Select the property in which you want to see search results and click property settings.<\/p>\n

          Scroll down and click on the adjust search console button.<\/p>\n

          <\/p>\n

          If your website is verified with search console, you should see your site URL. In case you don\u2019t, you might need to verify your site with google search console first.<\/p>\n

          Select the view in which you want the data to appear<\/p>\n

          <\/p>\n

          Click Save<\/strong>.
          To view search console results navigate to Acquisition >> Search Console<\/strong>.<\/p>\n

          <\/p>\n

          Few things to remember:<\/strong><\/p>\n

            \n
          1. The data in search console remains for the next 90 days, so the search console reports in analytics also include data from the past 90 days.<\/li>\n
          2. The data search console collects from search results will appear in analytics after 48 hours. So if you want to see the latest data to the date, go to your search console.<\/li>\n<\/ol>\n

            You can understand more about using search console reports here<\/a>.<\/p>\n

            \n 6. Tracking Site Search<\/b>
            \n<\/h2>\n

            The beauty of google analytics is that you can track anything on your site, even a search bar. Using site search data you can:<\/p>\n

              \n
            1. \n

              Get ample amount of long tail keywords<\/p>\n<\/li>\n

            2. \n

              Come up with new content ideas<\/p>\n<\/li>\n

            3. \n

              Get to know what your audience is searching for<\/p>\n<\/li>\n

            4. \n

              Improve your site\u2019s SEO<\/p>\n<\/li>\n<\/ol>\n

              To set up site search in google analytics, follow the steps below:<\/p>\n

                \n
              1. \n

                Sign in to your analytics account and select the view in which you want to see the search results.<\/p>\n<\/li>\n

              2. \n

                Go to view settings<\/p>\n<\/li>\n

              3. \n

                Enable the site search tracking<\/p>\n<\/li>\n

              4. In the query parameter box, add the words which belong to an internal query parameter. Query parameters might be categories, terms or tags on your site that are attached to the URL when searched. You can check for these queries and add upto 5 of them in the box.<\/li>\n
              5. Select the check box if you want to strip query parameters from the URL. <\/strong><\/li>\n<\/ol>\n

                <\/p>\n

                \u00a0<\/p>\n

                       6. Click save to finish.<\/p>\n

                \n 8. Aka Annotations<\/b>
                \n<\/h2>\n

                Annotations in google analytics are powerful yet the most forgotten feature. This helps you keep notes within your analytics reports and understand spikes in traffic and unusual changes.<\/p>\n

                <\/p>\n

                Types of Annotations<\/strong><\/h3>\n

                Shared Annotations: <\/strong><\/p>\n

                      Is visible to any other person having access to the reports.<\/p>\n

                Private Annotations: <\/strong><\/p>\n

                      Only visible to you.<\/p>\n

                \u00a0<\/p>\n

                How to add annotations <\/strong><\/h3>\n

                To add an annotation open the report containing the line chart.<\/p>\n

                Click on the small button with an inverted triangle at the bottom of the chart<\/p>\n

                <\/p>\n

                click on the create new annotation button.<\/p>\n

                <\/p>\n

                Now you can keep notes about important events and create as many annotations as you need.<\/p>\n

                To enable this feature in analytics, navigate to the property in which you want to see your audience demographic data.<\/p>\n

                From below the property menu, click on property settings and scroll down to Advertising features and switch on the demographic and interest reports button.<\/p>\n
