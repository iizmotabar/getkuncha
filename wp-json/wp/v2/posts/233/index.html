Have you ever wanted to understand what journey your users are taking to get to a specific page? This previously needed creating a funnel and waiting for it to collect data. Using a hidden report in GA we can now do for any page and that too for historic data!<\/span><\/p>\n

Read on to see how we can use Google Analytics user flow reports to build funnels for any page.<\/span><\/p>\n

First things first: What is a funnel?<\/strong><\/h2>\n

A funnel in google analytics is a path a user takes to complete a specific action. You define this path to see if the visitors on your site follow the steps and at which step most of the visitors drop.<\/span><\/p>\n

At first, it is difficult to define the specific path but once you have enough data in your analytics, you can do this easily. <\/span><\/p>\n

How to Create Any Funnel Using User Flow Reports?<\/strong><\/h2>\n

To help you better understand the process, I will explain the steps using an example. <\/span><\/p>\n

Let\u2019s say you want to define steps and create a funnel for your contact us page. The funnel would include the previous pages most visitors viewed to reach this page. <\/span><\/p>\n

Note: <\/em>Make sure to select a larger date range to get more data.  <\/em><\/p>\n

Follow the steps below to create this funnel.<\/span><\/p>\n

Step#1: <\/strong>view the user flow reports by landing pages.<\/span><\/p>\n

view reports by landing pages<\/p><\/div>\n

Step#2: <\/strong>click on the page(\/contact) which you want to track and select \u2018explore traffic through here\u2019. <\/span><\/p>\n

explore traffic through here<\/p><\/div>\n

The next page will display the specific page along with the previous and next pages. <\/span><\/p>\n

explore specific page<\/p><\/div>\n

From the above report, you can do three important things. <\/span><\/p>\n

1. Define a page group<\/strong><\/p>\n

Click on the edit icon and specify the page or the group of pages which you want to track. For example, your blog page may contain different pages. You can group the top pages to view the funnel.  To create a funnel for two different pages you have to select a <\/span>regular expression<\/span><\/a> from the match type drop-down.   <\/span><\/p>\n

define page group<\/p><\/div>\n

Next, enter the regular expression and click on apply button to view user flow map. <\/span><\/p>\n

2. View the previous page<\/b><\/p>\n

Click on the \u2018+step\u2019 button to view the previous pages viewed by the user to reach this page.<\/span><\/p>\n

view previous pages<\/p><\/div>\n

Once you selected previous steps you can understand the specific path most <\/span>users take to reach this page. From the above report, you can see, most of the<\/span>    <\/span>users enter to the contact page from home page, services page and work. <\/span><\/p>\n

To view a clear path, highlight the previous page by clicking on it and select \u2018highlight traffic through here\u2019.<\/span><\/p>\n

Now you can see clearly which path most users took to reach the contact page. From this report you can define a funnel to be <\/span>home >> about >> services >> contact<\/strong>. <\/span><\/p>\n

highlight traffic<\/p><\/div>\n

The same way you can highlight the other pages to create a different funnel. <\/span><\/p>\n

3. View the next page<\/b><\/p>\n

By clicking on the next step button, you can view the next pages a user viewed after viewing the work page. This can help you understand other pages viewed    <\/span>by the same user and page where they exit.<\/span><\/p>\n

view next pages<\/p><\/div>\n

How to use this?<\/strong><\/h2>\n

Once you have created the funnel, the next step is to understand how to use these funnels. How you analyze the user behavior and improve the funnel to reduce drop offs. Below are the two important things you can do with user flow funnels. <\/span><\/p>\n

Find where user Goes after seeing a specific page?<\/strong><\/h3>\n

To do this, add the page you want to start the funnel with as step 0. Use the View Next Page<\/a> option and see what do people do after getting to a page. Do they move on to a desired page? Exit or go somewhere else? You click +Step -> multiple times to continue building the path.<\/p>\n

How do people get to a Key page?<\/strong><\/h3>\n

Want to find how users get to a desired page (maybe the thank you page)? Add the desired page as step 0 and use the view previous step<\/a> to see where they started the journey.<\/p>\n

You click <\/span>+ Step <-<\/strong> multiple times to continue building the path backward.<\/span><\/p>\n

This is a super useful feature when trying to quickly understand user paths or see which pages are used together by users. You can also double the fun by adding advanced segments based on traffic source \/ user type and improve analysis even further.<\/span><\/p>\n
