In our recent article, we outlined a brief introduction to Mixpanel<\/a>. It is a good start for anyone thinking of using Mixpanel or using it.<\/p>\n

<\/p>\n

Insights – Deeply Analyze Users & Actions<\/strong><\/h2>\n

Insights is a powerful Mixpanel feature that allows you to answer complex questions. It allows you to visualize your data in multiple ways using different data visualization and functionalities.<\/p>\n

Using insights, you can create an advanced report for your marketing team. Using the query builder you can ask complex questions and also compare users and events.<\/p>\n

To get into insights, click on the activity analysis drop-down from the top menu and click insights.<\/p>\n

<\/p>\n

Inside the insights interface, you will see the list of all the events you have captured and a query builder on top.<\/p>\n

<\/p>\n

Query builder is the actual tool you will be playing with to dive deeper into users and events. You can use it to ask anything about your users and events. So if you look at the above image, I have currently three events recording pageview, link click and form submission.<\/p>\n

I can use the query builder to compare the total number of events vs unique events. Let\u2019s create a report to compare all pageview events vs total pageview events.<\/p>\n

Click on \u2018your top events\u2019 drop-down and select pageview event<\/p>\n

Click the next plus(+) icon and again select the pageview event from the drop-down<\/p>\n

From above the second pageview events selected, click on show total drop-down and select unique.<\/p>\n

Finally, select the date range to compare the events. By default, it compares with the last 96 hours, but you can change this by clicking on the time drop-down.<\/p>\n

<\/p>\n

This was a very simple way of comparing unique pageview vs total pageviews. Let\u2019s create a more advanced report to understand which devices were used and which pages were mostly viewed within a given date range.<\/p>\n

Let\u2019s look at the below animation to create the desired report.<\/p>\n

<\/p>\n

Once you create a report, you can save it to share with your team, add it to the dashboard or download as CSV file.<\/p>\n

To understand more about insights, click here<\/a>.<\/p>\n

Segmentation – Measure People’s Action<\/strong><\/h2>\n

Segmentation allows you to drill down events by its properties. You can use segments to break down a single event to see how many times it has been performed over a given date range.<\/p>\n

Let\u2019s take our previous example of pageview event to see the number of times pageview event was performed across browsers in each hour.<\/p>\n

First, you will have to select the event and break down by browser and filter it by hour.<\/p>\n

<\/p>\n

The above image displays the number of times pageview event occurred on each browser by hour.<\/p>\n

Let\u2019s create an advanced segment to see the total number of pageview events happened in Pakistan on android devices.<\/p>\n

<\/p>\n

You can then bookmark this segment or add to the report.<\/p>\n

The segmentation chart can also be used for comparison. \u00a0For example, you can compare multiple browsers, cities, countries and any other property that is available.<\/p>\n

Let\u2019s select the pageview event and segment by browser.<\/p>\n

<\/p>\n

Scroll down to the chart section and click on the top right button of the chart. From the drop-down select the browsers you want to compare as displayed in the gif below.<\/p>\n

<\/p>\n

Another important feature that is useful in segmentation chart is annotation. This feature helps you add notes to the chart whenever you make any update on your product. For example, you have added a CTA button on November 26, 2017. You can add a note about this change on the chart to see the changes that occur from that day onwards.<\/p>\n

To add an annotation, hover over the data point on the graph and click to add the note.<\/p>\n

<\/p>\n

Although, segmentation and Insights look similar but have many differences that make them separate. One of the main difference is in the query builder. Insights allow you to compare multiple events and properties in a single chart. It also contains multiple types of charts to view data from a different point of view.<\/p>\n

To understand the difference between insights and segmentation<\/a>, try to use both the features.<\/p>\n

Live View – Live Streaming<\/strong><\/h2>\n

Live view displays the current events being performed on your website. It refreshes every time a new event occurs. It displays the list of events happening on your site along with properties.<\/p>\n

You can click on the event to view the properties that is associated with each event. You can view the properties you have assigned and those assigned by Mixpanel.<\/p>\n

<\/p>\n

You can also apply filter the live view report by event and properties. To do this, click on the filter icon on the top left of the report and select the event\/property.<\/p>\n

<\/p>\n

Click on the add button to apply the filter. Once you apply this filter you will not see any data you have filtered out in real-time reports.<\/p>\n

The activity analysis feature empowers you to dive deeper into user actions and understand how a user interacts with your website.
