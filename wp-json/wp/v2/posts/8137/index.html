We already know that google analytics is specially developed for tracking external visitors apart from you and people under your company. Excluding internal traffic is one of the essential steps in configuring google analytics<\/a>.<\/p>\n

This article will help you exclude single and multiple IPs from your analytics account using multiple methods.<\/p>\n

How to Exclude A single IP Address<\/strong><\/h2>\n

1. Select the view for which you want to apply filter and then go to filters and click add filter<\/p>\n

2. Enter the filter name as \u201cexclude internal traffic\u201d<\/p>\n

3. Select the filter type as predefined<\/p>\n

4. Enter your IP address as displayed in the image below<\/p>\n

<\/p>\n

Alternate Option: Exclude Single IP<\/strong><\/h2>\n

There is another way you can use to exclude traffic from your own browser. To do this, install Da Vinci Tools<\/a> chrome extension. once you install, open your Google Analytics account.<\/p>\n

you will then see a switch on the top right of your account along with your tracking ID.<\/p>\n

<\/p>\n

when you turn the toggle off, this extension will block your hits going to GA.<\/p>\n

This is a good solution for testing purposes and also for excluding internal traffic. once you add this extension, it will work for all properties in your analytics account. you will not have to add separate filters for each account\/property.<\/p>\n

How to Exclude Multiple IP addresses<\/strong><\/h2>\n

Everything is similar to how we did for a single ip address above except, you have to enter regex for multiple ip addresses. Let\u2019s say we have two exclude two different ip addresses 192.168.0.1 and 192.168.1.1. The regex would be<\/p>\n

^192\\.168\\.0\\.1$|192\\.168\\.1\\.1$<\/p>\n

Start with \u2018^\u2019 then precede each dot with a backslash(\\) and put \u2018$\u2019 at the end of ip and separate multiple ips with a pipe(|). The easiest way to write regex for multiple ips is first write down the ips together<\/p>\n

1. 192.168.0.1192.168.1.1<\/p>\n

2. Add \u2018^\u2019 before the first ip address ( ^192.168.0.1192.168.1.1)<\/p>\n

3. Add pipeline between both ip addresses (^192.168.0.1|192.168.1.1)<\/p>\n

4. Add backslash just before every dot (^192\\.168\\.0\\.1|192\\.168\\.1\\.1)<\/p>\n

5. add \u2018$\u2019 sign at the end of each ip address (^192\\.168\\.0\\.1$|192\\.168\\.1\\.1$)<\/p>\n

You can add multiple ips using the above regex.<\/p>\n

<\/p>\n

How to exclude dynamic IP addresses<\/strong><\/h2>\n

One of the most easiest way to exclude traffic from multiple dynamic ip addresses is to install google analytics opt-out addon<\/a>. This addon tells the javascript not to send the pageview hit.<\/p>\n
