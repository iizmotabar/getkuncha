What is Google Tag Manager?<\/strong><\/h2>\n

Google Tag Manager is a FREE tag management solution put together by Google. It is used to deploy analytics and marketing tags on websites and mobile apps. It is best developed for marketers to manage website tags without changing site codes.<\/p>\n

Benefits of Using Google Tag Manager<\/strong><\/h2>\n

Although there are a lot of benefits of using Google Tag Manager but the most important is the way you can easily install any analytics or marketing tag. It can be used to install Google Analytics, Facebook Pixel, Mixpanel, Segment and many more.<\/p>\n

I will not explain the benefits in details because we have already published a list of things Google Tag Manager can do<\/a>.<\/p>\n

Getting Started with Google Tag Manager<\/strong><\/h2>\n

As mentioned earlier, GTM is a FREE tool so you don\u2019t have to worry about any payment. To use GTM, you will have to create an account<\/a><\/strong> with your Gmail id and add GTM code to your website.<\/p>\n

Google Tag Manager provides two types of codes to add on your website. One in the head section before <\/head> tag and the other in the body section right after the opening body tag <body>.<\/p>\n

<\/p>\n

Once you add both the codes on your website, you can then test with Google Tag Assistant<\/a> to verify installation.<\/p>\n

The Components of Google Tag Manager<\/strong><\/h2>\n

There are three main components of Google Tag Manager. Tags, Triggers, and Variables. We will understand each in detail because you will be mostly working with them.<\/p>\n

Tags:<\/strong><\/h3>\n

As we know that GTM is used for managing codes on your website, that is why it is known as tag manager. A tag in GTM is any code snippet that you add to your website. It can be a Google Analytics tag, Adwords tag, Facebook Pixel tag or any other. In GTM language, we call them as tags.<\/p>\n

GTM contains built-in templates for a lot of analytics and marketing tools. Templates make it possible to add a tag easily. If you want to add a tag on your website, for which a template is not available, you can add it via custom HTML tag.<\/p>\n

To view the list of available tag templates, click on Add a New Tag on the home page.<\/p>\n

<\/p>\n

Next, click on tag configuration<\/p>\n

<\/p>\n

You can scroll the pop-up list to view all the templates available. You will also see a custom HTML tag, which is used to add any JavaScript code on your site.<\/p>\n

<\/p>\n

Triggers:<\/strong><\/h3>\n

A trigger in GTM is a condition that evaluates to true or false during runtime. It observes any change on the site and decides whether or not to fire a tag.<\/p>\n

Google Tag Manager offers a list of trigger types to fire your tag. Some of the common and mostly used triggers are listed below:<\/p>\n

Pageview: <\/strong>fires a tag when a pageview occurs<\/p>\n

All Elements: <\/strong>fire a tag on all clicks<\/p>\n

Just links: <\/strong>fire a tag when a link is clicked<\/p>\n

Form Submission: <\/strong>fires a tag when a form is submitted<\/p>\n

YouTube Video: \u00a0<\/strong>fires a tag when a YouTube video is played<\/p>\n

JavaScript Error: <\/strong>fires when JavaScript error occurs<\/p>\n

To view the trigger types available in GTM, navigate to triggers from the left column and click new.<\/p>\n

<\/p>\n

Click trigger configuration from the pop-up window. Scroll down the trigger types list to view all the triggers available in GTM.<\/p>\n

<\/p>\n

A trigger must be associated with a tag to execute. To understand more about triggers in depth, you can follow this article<\/a>.<\/p>\n

Variables:<\/strong><\/h3>\n

A variable in Google Tag Manager is a name-value pair for which the value is generated during runtime. A variable will remain empty in your GTM unless it is associated with a tag, trigger or any other variable.<\/p>\n

There are two types of variables in Google Tag Manager.<\/p>\n

Built-in variables and user-defined variables.<\/p>\n

Built-in Variables<\/strong><\/h3>\n

Built-in variables are predefined in Google Tag Manager. We can not customize them. Although they are disabled by default. To view the list of built-in variables available in GTM, click on Variables from the left column.<\/p>\n

<\/p>\n

Click on the configure button to view and enable any of the built-in variables.<\/p>\n

Let\u2019s look at the page variables:<\/p>\n

Page URL: <\/strong>returns a string with full URL of the current page (https:\/\/example.com\/how-to-test)<\/p>\n

Page Hostname: <\/strong>returns a string with the full hostname of the current URL(https:\/\/example.com)<\/p>\n

Page Path: <\/strong>returns string with relative path of the current page(\/how-to-test)<\/p>\n

Referrer: \u00a0<\/strong>returns a string with the full URL of the page that brought the user to the current page(https:\/\/test.com\/blog)<\/p>\n

User-Defined Variables:<\/strong><\/h3>\n

These variables are defined by users and not pre-defined. You can create your own variable to save data during run-time. For example, you can create a user-defined variable to save the Google Analytics tracking ID or you can pass a form field value via user-defined variables. You can create your own variable under pre-defined variables.<\/p>\n

To understand further about GTM variables, you can the variable guide for Google Tag Manager<\/a>.<\/p>\n

Testing and Debugging in Google Tag Manager<\/strong><\/h2>\n

The beauty of Google Tag Manager is its testing and debugging features. Whenever you make a new change in GTM, you can test before making them.<\/p>\n

For testing and debugging, we use the preview mode and to publish the changes we simply click on the submit button.<\/p>\n

In your GTM account, you will see the preview and submit buttons on top right of your account.<\/p>\n

<\/p>\n

Whenever you add anything in tag manager, it is recommended that you preview and test changes. Let\u2019s say you have added Google Analytics tag in GTM and want to preview before publishing it live.<\/p>\n

Click on the preview button from the top right of your GTM account.<\/p>\n

When the preview mode load, open your website in a new tab.<\/p>\n

After the website has loaded, you will see the debug mode at the bottom of the page.<\/p>\n

It displays the list of tags that are fired and those aren\u2019t fired.<\/p>\n

<\/p>\n

As you can see in the above image, our Google Analytics tag(GA Pageview) has fired successfully. You can also see the type of tags we can install with GTM(Facebook, Mixpanel, Optimize, Adwords..)<\/p>\n

Once you confirmed that tag is firing, you can then click on the submit button to publish changes.<\/p>\n

Adding Multiple Users<\/strong><\/h2>\n

In Google Tag Manager, you can add other people to work on your container. For example, if you have hired a tag manager expert to work for you, you can add him\/her as a user.<\/p>\n

This way you will not have to share your Gmail credentials to anyone else for the sake of Google Tag Manager.<\/p>\n

You can restrict the user from making changes in GTM by controlling the access level. A user can read, edit, approve and publish changes in a container.<\/p>\n

To read more about users and permissions click here<\/a>.<\/p>\n

Import & Export Container<\/strong><\/h2>\n

Importing and exporting containers in tag manager allows you to create or share your implementation with others. When you import or export a container, it is actually a JSON file containing tags, triggers, and variables.<\/p>\n

One of the main reason you would want to import or export a container is, you may have multiple similar websites where you want to set up similar tags, triggers, and variables.<\/p>\n

To learn how to import and export a container in GTM, follow our step by step guide<\/a>.<\/p>\n

Folders – Organize Tags, Triggers & Variables<\/strong><\/h2>\n

Google Tag Manager seems to be empty at first and easy to handle fewer tags, triggers, and variables. By adding more tags and triggers, your container can into a complex thing.<\/p>\n

To avoid the complexity and easy to work for yourself and for your team, you can organize them into logical groupings called folders.<\/p>\n

Always use folders to organize tags, triggers, and variables<\/a>.<\/p>\n

Conclusion<\/strong><\/h2>\n

Google Tag Manager can make your life a lot easier. You can improve your site speed by merging all the tags in tag manager and firing them specific pages only.
