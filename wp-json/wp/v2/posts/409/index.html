Mixpanel is quite different from Google Analytics. It offers different types of libraries for setting up Mixpanel on different platforms like websites or mobile apps. \u00a0For websites, Mixpanel offers a JavaScript library which contains multiple functions to set up events.<\/p>\n

For example, if you want to track pageviews, you will have to setup pageview event using the pageview function(mixpanel.track(\u201cPageview);. So you don\u2019t need to be a ninja but at least need to have basic knowhow of different types of Mixpanel functions to properly follow your tracking plan.<\/p>\n

The Mixpanel tracking snippet initiates the Mixpanel library which contains the mixpanel.init function. If you have enabled auto tracking, this will track pageviews automatically, otherwise, you will have to add the pageview function(mixpanel.track(\u201cPageview\u201d);).<\/p>\n

In this article, I will walk you through some of the basic and necessary functions that are used to track events in Mixpanel. This will help you understand how Mixpanel events can be used for tracking different types of events. You can,\u00a0of course, thank me later.<\/p>\n

Intro to Mixpanel<\/strong><\/h2>\n

As you know that Mixpanel is an event-driven advanced analytics tool. It focuses on tracking events rather than pageviews and we already have covered intro to Mixpanel<\/a> in our previous posts.<\/p>\n

So for tracking events, Mixpanel provides a certain set of libraries each with different functionality. We have to use these libraries to setup everything we need to track.<\/p>\n

Basic Functions to Set up Mixpanel<\/strong><\/h2>\n

Below is a list of functions to use while setting up Mixpanel tracking.<\/p>\n

mixpanel.init<\/strong><\/p>\n

As the name suggests, we use this function to initiate the mixpanel tracking<\/p>\n

mixpanel.init(‘new token’, { your: ‘config’ }, ‘library_name’);<\/p>\n

mixpanel.track<\/strong><\/p>\n

This lets you stalk your users\u2019 activity, not literally though. mixpanel.track used to track events and properties performed by the user<\/p>\n

mixpanel.track(‘Signup’, {‘$gender’ : \u00a0‘Male’, \u00a0‘$age’ : \u00a021});<\/p>\n

mixpanel.register <\/strong><\/p>\n

This is used to store super properties i.e properties that we want with all events and don\u2019t have to send them again to mixpanel.<\/p>\n

mixpanel.register({ \u00a0‘$email’ : \u00a0‘jdoe@example.com’, \u00a0‘account_type’ : \u00a0‘Free’ });<\/p>\n

mixpanel.people.set<\/strong><\/p>\n

This lets you set the people, again not quite literally. mixpanel.people.set is used to set properties on the user profile<\/p>\n

mixpanel.people.set({ ‘$company’ : \u00a0‘Acme’, \u00a0‘plan’ : \u00a0‘Premium’ \u00a0});<\/p>\n

mixpanel.identify<\/strong><\/p>\n

With mixpanel.identify you can identify people. This is used to identify and create a user profile<\/p>\n

mixpanel.identify(“YOUR_USER_ID”);<\/p>\n

mixpanel.alias<\/strong><\/p>\n

This brings past to present. mixpanel.alias is used to stitch data between user\u2019s previous and current id<\/p>\n

mixpanel.alias( ‘newer_id’, ‘new_id’ );<\/p>\n

mixpanel.register_once<\/strong><\/p>\n

This is similar to the register function with the exception that once set, it can\u2019t be overwritten<\/p>\n

mixpanel.register_once({ ‘signup_date’ : \u00a0new Date() });<\/p>\n

mixpanel.people.set_once<\/strong><\/p>\n

This is similar to the people.set function with the exception that once set it can\u2019t be overwritten<\/p>\n

mixpanel.people.set_once({ ‘signup_date’ : \u00a0new Date() });<\/p>\n

mixpanel.people.increment<\/strong><\/p>\n

This is used to maintain counts on users profiles.<\/p>\n

mixpanel.people.increment( ‘page_views’, \u00a01 );<\/p>\n

<\/p>\n

The functions we discussed above are enough to implement a basic tracking plan. But let\u2019s dig a little deep in the Mixpanel library and see what additional items Mixpanel can offer us.<\/p>\n

mixpanel.init<\/strong><\/p>\n

We use this function to initiate the Mixpanel tracking. We can also pass configuration with the init to override the default configs like debug or persistence. We can also run multiple Mixpanel projects by declaring the library name.<\/p>\n

mixpanel.get_property<\/strong><\/p>\n

We can use super properties stored in the Mixpanel cookie with help of this function<\/p>\n

mixpanel.cookie<\/strong><\/p>\n

Cookies are scrumptious. Mixpanel.cookie contains all the data stored in Mixpanel\u2019s private cookie including the name of the cookie and super properties.<\/p>\n

mixpanel.get_distinct_id()<\/strong><\/p>\n

Returns the unique id that is used to store the user profile in Mixpanel.<\/p>\n

mixpanel.getConfig<\/strong><\/p>\n

Returns the value of the config for Mixpanel, for example, the token, disable persistence, cross_subdomain_cookie.<\/p>\n

mixpanel.setConfig<\/strong><\/p>\n

Update the default configuration of Mixpanel. For example, we want to update the cookie life or update the blacklist properties or to enable Mixpanel debug function<\/p>\n

mixpanel.config<\/strong><\/p>\n

Contains all the config data for Mixpanel<\/p>\n

mixpanel.disable<\/strong><\/p>\n

Doesn\u2019t track events passed in the param for the user, if no params, then all events won\u2019t be tracked<\/p>\n

mixpanel.people.delete_user<\/strong><\/p>\n

This helps you with moving on. mixpanel.people.delete_user function deletes the user\u2019s entire data from Mixpanel<\/p>\n

mixpanel.time_event<\/strong><\/p>\n

Tracks the elapsed time between the \u2018time event\u2019, when called and \u2018track calls\u2019<\/p>\n

mixpanel.people.clear_charges<\/strong><\/p>\n

It helps you in case of regrets and permanently clears all revenue report transactions from the current user\u2019s people analytics profile. We should use this to clear data from our test users.<\/p>\n

mixpanel.reset<\/strong><\/p>\n

Clears super properties and generates a new random distinct_id for the instance. Useful for clearing data when a user logs out<\/p>\n
