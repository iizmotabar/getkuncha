If you are looking to install Shareaholic via google tag manager, you have made the right decision. Because doing this will improve your site speed and allow you to control the plugin without having to touch your website code. <\/p>\n

Without wasting time on less important things, let\u2019s get started with the actual process. Follow the steps below to install Shareaholic via Google Tag Manager. <\/p>\n

Copy Shareaholic Site ID<\/strong><\/h2>\n

Step1:<\/strong> Create your Shareaholic account and connect your website. If you already have Shareaholic installed on your site but not via tag manager, check the code and remove the code snippet. <\/p>\n

Step2:<\/strong> login to your Shareaholic account and click on my sites drop down and select the website. <\/p>\n

<\/p>\n

Step3:<\/strong> Copy your site ID from below the left menu. <\/p>\n

<\/p>\n

Install Shareaholic via Google Tag Manager<\/strong><\/h2>\n

Note: If you are new to Google Tag Manager make sure you have enabled all the built-in click variables in GTM. You can do this by navigating to variables, under built-in variables click configure and check all the click variables.<\/strong>
Step1:<\/strong> Now, open a new tab in your browser and sign in your tag manager account.
Step2:<\/strong> select the container in which you want to add Shareaholic.<\/p>\n

<\/p>\n

Step3:<\/strong> inside your container, create a new tag. Click on the New Tag button.<\/p>\n

<\/p>\n

Step4:<\/strong> Enter the tag name as \u201cShareaholic\u201d and select the tag type as \u201cShareaholic\u201d. <\/p>\n

<\/p>\n

Good Read: Complete Guide to Google Tag Manager Events<\/a><\/p>\n

Step5:<\/strong> Enter the Shareaholic Site ID which you have copied earlier. Click on advanced settings and select tag firing option as once per page. <\/p>\n

<\/p>\n

Step6: <\/strong>Decide on which pages to display the shareaholic plugin. Let’s say you want to add this to your blog pages only. click on the triggering area below the tag configuration settings. <\/p>\n

Click on the plus icon on the top right of the page. <\/p>\n

<\/p>\n

Enter the trigger title as “Blog pages only”<\/p>\n

click on the trigger configuration and select the trigger type as page view.<\/p>\n

choose the trigger to fire on some pages<\/p>\n

Now you have to tell the condition. To display the plugin on blog pages add the condition as “Page URL contains \/blog\/”. Set the condition according to your URL type.<\/p>\n

<\/p>\n

Note: <\/strong>If you want to run the plugins to some specific pages set the condition as “page url matches regex” and enter regex<\/a> for the pages. <\/p>\n

Save the trigger. <\/p>\n

Step7:<\/strong> Save this tag. Preview and debug<\/a> container changes and then submit to see the changes live on your site. <\/p>\n

Now, you can view your website to check if the Shareaholic icons are hanging somewhere. <\/p>\n
