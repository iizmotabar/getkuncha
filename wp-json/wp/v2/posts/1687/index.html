URL Query parameters are variables that contain information and are attached at the end of URL preceded with a question mark(?). The query parameter might be a search string or any other parameters that are attached to pass some information. This guide will help you exclude all the URL query parameters from your Google Analytics reports.<\/p>\n

\u00a0<\/p>\n

URL Query Parameter Problem<\/strong><\/h2>\n

Let\u2019s take an example of a single URL to help you understand the actual problem. For example, we have a URL that is<\/p>\n

https:\/\/marketlytics.com\/analytics-faq\/exclude-url-query-parameters<\/strong><\/p>\n

Someone lands on our analytics-faq<\/a> page and enters the term \u201cquery parameters\u201d into the search bar. He clicks on the listed options and opens the relevant page. But this time the URL would be<\/p>\n

https:\/\/marketlytics.com\/analytics-faq\/exclude-url-query-parameters?rq=query%20parameter <\/strong><\/p>\n

The query string got attached to the URL.<\/p>\n

Now, Google Analytics will detect a unique pageview for this URL. So instead of displaying a single URL, Google Analytics displays two unique pageviews and you will see both the URLs listed in the reports.<\/p>\n

But we want Google Analytics to detect one unique pageview for both the URLs. For doing this, we will exclude the query parameters from the end of URL.<\/p>\n

Good Read: Excluding Multiple IPs in Google Analytics<\/a><\/p>\n

How to Exclude URL Query Parameters<\/strong><\/h2>\n

To exclude certain string parameters when processing we will enter the junk parameters into the \u201cexclude URL query parameters field\u201d under the view settings.<\/p>\n

To do this, navigate to Admin >>  View >> View Settings >> Exclude URL Query Parameters<\/strong><\/p>\n

<\/p>\n

If you have more than one parameter, enter each of the parameter separated by a comma(,). Here is an example:<\/p>\n

rq, axm, var<\/p>\n

Note: Remember that these queries will be excluded from the specific view, not all the views.<\/strong><\/p>\n

\u00a0<\/p>\n

How to Find Query Strings<\/strong><\/h2>\n

If you don\u2019t know which query strings to exclude or want to check for the list of strings, follow the steps below:<\/p>\n

Goto all pages report(Behavior >> Site Content >> All Pages)<\/p>\n

In the search bar enter \u201c\\?\u201d and hit enter<\/p>\n

<\/p>\n

This will display all the URLs containing query strings<\/p>\n

<\/p>\n

Once, you add the query parameters this will affect all the reports in the respective view. Also, this will not be applied on your historic data but will be applied to all the future data.<\/p>\n

One more thing to make clear here from the above picture. If your URL is<\/p>\n

\/blog\/?offset=1475847907418<\/p>\n

In the query parameter field, you will just enter the string \u201coffset\u201d that is anything between the question mark and the equality sign, not everything that is followed by a question mark.<\/p>\n

Note: Make sure you are not excluding any strings that carry some useful information and only junk parameters are added to the list.<\/p>\n
