When you become aware that your website is getting visitors, the next question you almost immediately want to ask is, \u201cwhat are they doing?\u201d. Which sections of your website are catching their fancy, what areas are they lingering on and what actions are they choosing to take? Tracking user activity can be fascinating but it can be also a difficult call to make at times.<\/p>\n

For once, when you\u2019re tracking user actions in Google Analytics, you have to make a choice – the choice between event tracking vs virtual pageviews<\/strong> as a way to track visitor interaction on your website. They both have their pros and cons but it\u2019s only after you get to know what they are about and can you really decide what\u2019s a better way to track user actions.<\/p>\n

And we give you a concise overview below:<\/p>\n

Pros and Cons of Event Tracking vs Virtual Pageviews<\/h2>\n

Event Tracking<\/strong><\/h3>\n

event tracking<\/p>\n

Event Tracking is a method that allows you to record any user interaction that happens on your website. It considers specific user actions as \u201cevents\u201d which are tracked. The user interaction could be in the form of  a video playback, a form submission or even just a click on a button. The complete guide to  Event Tracking in Google Analytics<\/a> will help you set up and understand the basics.<\/p>\n

The Good<\/strong><\/p>\n

    \n
  • Easy to setup and use.<\/li>\n
  • Can be configured to not affect bounce rate when an event occurs.<\/li>\n
  • Doesn\u2019t affect pageview count and is instead tracked separately.<\/li>\n
  • Google Analytics shows detailed report under Behaviour -> Events<\/strong> for all the events created.<\/li>\n
  • Three levels of data is collected: Category, Action and Label – all of which give you additional details.<\/li>\n<\/ul>\n

    The bad<\/strong><\/p>\n

      \n
    • Can\u2019t be used to track user navigation over different pages by setting up goal funnels in standard google analytics.<\/li>\n
    • Setting up events is easy, but require a strict naming convention when setting up several events to get relevant results.<\/li>\n
    • Google Analytics doesn’t show events data in Audience reports.<\/li>\n<\/ul>\n

      Virtual Pageviews<\/strong><\/h3>\n

      virtual pageviews<\/p>\n

      Virtual Pageviews is just  a standard google analytics pageview but rather than being triggered when a user navigates to a page, it is instead triggered on a particular user interaction similar to \u201cEvent Tracking\u201d. This shows up in All Pages report with your specified url. For using and debugging read the complete guide to virtual pageviews in google analytics<\/a>.<\/p>\n

      The Good<\/strong><\/p>\n

        \n
      • Can track user navigation between site elements and actual pageview by configuring virtual pages as in goal funnels. This opens various other information including reverse goal path, goal flow and page levels.<\/li>\n
      • It allows you to treat any action as page. Examples include: One step checkout, tabbed browsing, single page sites etc.<\/li>\n
      • It allows the use of page value to see relative value of various actions that may attribute to conversion.<\/li>\n<\/ul>\n

        The bad<\/strong><\/p>\n

          \n
        • The virtual pageviews are treated the same as rest of the website and Increase default pageview count (can be resolved by creating filtered view for virtual pages).<\/li>\n
        • Virtual pages are included in all pageview reports but no default report is created separating them, thereby inflating the pageview count.<\/li>\n
        • Identifier is required to be included in virtual pages to ease filtration from normal pages.<\/li>\n
        • Bounce rate is affected whenever a virtual pageview is triggered<\/li>\n<\/ul>\n

          In a nutshell:<\/strong><\/h3>\n

          As with most choices, this decision depends on what you require. In most cases, google analytics event tracking is the better choice due to its flexibility but if you have a single page site or a checkout flow that requires building a funnel, virtual pageview can help you better visualize user navigation.<\/p>\n
